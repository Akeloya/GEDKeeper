File encryption

GEDKeeper supports storing and loading encrypted GEDCOM files. The standard does not support encryption, therefore GEDKeeper chose its own implementation for file encryption.

When you are saving a database, you must choose "GEDKeeper encrypted GEDCOM files (*.geds)" as target file format. Files of this format have ".geds" extension. After you have selected the file format and entered file name, you are prompted to enter password. When you are about to load an encrypted file, you select the same file format and enter your password when you will be prompted.

Attention: GEDKeeper uses not the most cryptographically secure encryption method through existed ones. But following "security best practices" it does not store a password within an encrypted file. Therefore, if you will forget or lose your password once, nobody (including GEDKeeper author) will be able to open your database file. Be careful and enter your password cautiously. Any symbol inadvertently added can be a cause for losing your file.

Currently GEDKeeper uses "DES" algorithm for encryption and "SHA1" for signing. In case of need a better cryptography service may be used (for example, "3DES" or "AES").

Also, the following features are delayed for future development: choosing encryption method by user, prompting for password only once for the same file, adding password repeat field when a file is saved at first time (to avoid typos in password text).
