General

Files encoding

GEDKeeper can save GEDCOM-database using two encodings: ASCII/ANSI and UTF-8.

ANSEL encoding is not supported because it is deprecated and it was used mainly on USA region.

For better portability it is recommended to use UTF-8 encoding. UTF-8 has very best support for national symbols and it is compatible with Microsoft Windows and Linux platforms.

Work mode

There are two modes for working with parents' families in person modifying dialog: "Simple" and "Advanced". In simple mode you handle parents' family as a whole object, i.e. attach and detach the family as a whole. In advanced mode you can attach and detach mother or father from the person individually.

Beginners should start with simple mode because novices frequently make errors while they are modifying relations between parents and children. They take this link as a direct one, and do not take family itself into account, as a whole object used as real link between parents and children.

User interface

Highlight person with no parents or family

Highlighting persons with no parents or family improves navigation through the list. The following color scheme is applied: white background is used for persons with family and parents, flaxen background -- for persons without family; light red -- for persons without parents. Light colors do not prevent to get information.

Family tree

There are two setting groups for family tree generation: how persons are represented in the tree and how the tree appears on a render target.

Person representation in the tree

Here you define what data you will see in the tree: last name, first name and surname, birthday and obit, part of dates.

When "Relation degree" option is on, every person has visible relation degree, that is calculated on each tree rebuild relative to the central person in the tree.

"Additional symbols" shows special symbols for some categories of people. There are special icons for veterans of the World War II, died, veterans of Labor, cavalier of Cross of St. George, repressed. These categories are defined in the user-defined reference section in person editor.

"Embellished appearance" controls some additional graphical elements and shadow.

"Hide passed away in childhood" option becomes helpful when you handle several sources strongly, because it can reduce number of visible persons and make your task much simpler.

Appearance (color and font)

This setting group defines colors and the main font used in tree rendering.

Pedigrees

Here you set up pedigree generation. "Include person attributes" checks if person's life events will be included in pedigree. "Include notes" option checks if text notes from person's biography will be included in pedigree. "Include sources" option checks if pedigree includes source's number that references a person. There is the source list at the bottom of pedigree. "Format" option, that has two possible values: "Redundant" and "Conventional", controls level of details for person description printed in pedigree. Usually, you should prefer the conventional format, because the redundant one may contain too much unnecessary information.
