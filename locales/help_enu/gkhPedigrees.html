Pedigrees

GEDKeeper has ability to generate pedigree for generations. There are two pedigree schemes available: d'Aboville's and Konovalov's schemes. Both schemes make identical pedigrees with the only difference in persons numbering.

d'Aboville's scheme assigns #1 to the first man in the first generation; the man's first descendant in the next generation gets #1.1; grandson -- #1.1.1 and so on. This system is suitable when one has started to create tree draft on paper -- it is easy to add elements to any generation in the pedigree using d'Aboville's scheme.

Konovalov's scheme gives numbers like "X-Y" to all people, where "X" is sequence number for a man in the pedigree, and "Y" is number of the man's ancestor. This system provides an easy navigation but it is hard enough to manage Konovalov's scheme using paper medium -- adding new persons to existing generations will be a tough task; numbering get broken, the pedigree becomes a mess.

GEDKeeper sees no big differences between those two schemes and allows you to create pedigrees using any of the schemes. Choosing a scheme is based on user preferences (appearance of pedigree, and how to view and manage pedigree).

To create a pedigree using one of the scheme described above, select required entry in the person list, and find menu item with the scheme name in "Pedigree" menu. Or you can use button on the toolbar.

Pedigree is created as a PDF file with hyperlinks between persons inside. The created file is opened by a program registered in your system to open PDF files. Using such program, you may print the pedigree.

You can find settings for pedigree generation in the "Options" dialog on the "Pedigrees" tab.

  • When "Include person attributes" option is on, pedigree includes person's life events.
  • When "Include notes" option is on, pedigree includes text notes from person's biography.
  • When "Include sources" option is on, pedigree includes source's number that references a person. There is the source list at the bottom of pedigree.
  • "Format" option has two possible values: "Redundant" and "Conventional". This option controls detail level for person description printed in pedigree. Usually, you should prefer the conventional format, because the redundant one may contain too much unnecessary information.
