Filter masks

You use masks in GEDKeeper to select persons, events or locations having specific name. Text of a mask consists of general printable characters, allowed for object names, and two special symbols '*' and '?'.

Mask name is used because '*' and '?' symbols mask some arbitrary parts of a value.

  • '*' inside a pattern matches any number of any characters;
  • '?' matches one any character.

If you want to specify several different masks, you can use "|" symbol to delimit each mask. For example, to get all persons with names Mitch and John you can use "*Mitch*|*John*" mask. "|" symbol acts as "OR" conjunction, therefore any person whose name matches one of the masks, matches entire "Mitch*|John*" mask.

Any person's name, event or location always matches '*' mask. When you use such mask in a filter, it will show all objects.

When you need to get all persons with "Ivanovs" as last name, you use "Ivanov*" masks in "Name mask" filter. Star symbol matches any characters after initial substring "Ivanov". This way you get all persons with Ivanov, Ivanovskih and so on as their last name. If you want to filter out all persons with "Vasiliy" first name, you should use "* Vasiliy *" pattern; thus you are masking any last name and surname. To get all women with Vasilevna surname you use "* Vasilevna", masking any first name and last name.

Examples:
- Names "Litvin Ivan Andreevich" and "Semenov Ivan Andreevich" can be filtered out with "*Ivan Andreevich" mask;
- Any person with first name, last name or surname having "Ivan" inside, matches "*Ivan*" mask;
- Any woman with "Natalia" or "Natalya" name matches "Natal?a*" pattern.
